## 1. Vision & Market Validation

Forge is the world's first full-stack AI business foundry.
Anyone—from a non-coder to an enterprise CTO—describes an idea in plain language and gets a live, hosted, monetizable software product in minutes, accessible on desktop, mobile, or as a complete application.

### Market Reality (2026)

- Global vibe coding market: $4.7B (2026), projected $12.3B by 2027 (38% CAGR)
- 92% of US developers use AI coding tools daily
- 41% of all code globally is AI-generated
- Gartner: 60% of new code AI-generated by end of 2026
- 63% of vibe coding users are non-developers
- Cursor: $2B ARR, valuation $29.3B
- Lovable: $300M+ ARR, valuation $6.6B
- Emergent: $100M ARR in 8 months, 6M+ builders
- App Store submissions Q1 2026 up 84% YoY from vibe coding
- 21.7% enterprise preference for outcome-based pricing (parity with per-seat)

### Critical Gaps We Exploit

- **Cursor**: single-model, no deployment, no mobile, no token incentives
- **Claude Code**: terminal-only, Claude-exclusive walled garden, Mac-only, extreme token burn
- **Lovable**: opinionated stack (React+Supabase only), 35% gross margins, no enterprise governance
- **Bolt.new**: frontend-focused, limited backend, manual deployment
- **Replit**: cloud IDE, limited autonomy, no tokenized marketplace
- **OpenRouter**: pure API middleware — no IDE, no agents, no deployment
- **OpenClaw**: powerful agent runtime but no built-in business model, no marketplace

No platform unifies model-agnostic routing + agentic coding + one-click deploy + marketplace + tokenized ownership across desktop and mobile.

## 4. Database & Backend Hosting Strategy (3 Tiers)

### Tier 1: Forge Managed Backend

- Platform auto-provisions, optimizes, and manages backend on Forge's cloud.
- For simple apps: SQLite auto-managed; full-stack: PostgreSQL via Supabase; enterprise: dedicated DB clusters.
- User pays a 20–30% premium on underlying compute/storage for zero-DevOps.
- Best for non-tech founders, rapid prototyping. Pricing: $3–$50/mo.

### Tier 2: Bring Your Own Cloud (BYOC)

- Agents deploy databases directly into user's own AWS/GCP/Azure/DigitalOcean account.
- User owns infrastructure and billing; Forge fully automates setup and optimization.
- Data plane entirely in user's VPC, control plane on Forge. Essential for compliance.
- Platform fee $500–$5,000/mo depending on scale.
- Enterprise requirements: data never leaves VPC, IAM integration, audit-ready IaC.

### Tier 3: Self-Hosted / Bring Your Own Database

- Power users export code + schema as portable artifacts, deploy to any PostgreSQL-compatible host, or connect Forge agents to existing databases via connectors.
- Full control, zero dependency on Forge infrastructure.
- Forge open-core agent runtime and router are MIT/Apache 2.0 licensed; free.
- Monetized indirectly via premium support, consultancy, enterprise licenses.

**Supported Databases**: SQLite (default), PostgreSQL, MySQL/MariaDB, Microsoft SQL Server, MongoDB (via connector), cloud-native (RDS, Cloud SQL, Azure Database).

### Why This Is an Unfair Advantage

- Lovable: Supabase only
- Bolt.new: limited backend
- Replit: built-in only
- Cursor: BYO only
- **Forge: full spectrum — managed, BYOC, self-hosted**

Eliminates vendor lock-in fear. Users start on Forge Managed, migrate later without losing functionality.

## 5. Desktop & Mobile App Architecture

### Desktop App

- Native IDE (Eclipse Theia fork) with Agent Canvas, router dashboard, built-in terminal.
- **Computer Control**: cross-platform (Mac/Windows/Linux) unlike Claude Code (Mac-only). Capabilities: open/control GUI apps, simulate mouse/keyboard, take screenshots for verification, run build tools. Safety: app-level authorization, permission tiers (view-only/click-only/full), ESC to stop, terminal excluded from screenshots.
- **Forge Advantage**: routes computer-control tasks to cheapest capable model, saving 60–90% API costs.
- **KAIROS Daemon**: always-on, watches filesystem, auto-fixes, sends push notifications.
- **Local-first**: all agent memory local (SQLite/LanceDB), operates offline with local models, syncs later.

### Mobile App (iOS + Android)

- **Voice-first vibe coding**: "Build me a mobile app that..." → working app in minutes.
- Full lifecycle: create, iterate, preview, deploy, push updates.
- App Store publishing pipeline: auto-generate metadata, screenshots, submit for review.
- Push notification command center: agent task alerts, deploy health, approve decisions.
- Seamless desktop–mobile continuity: shared state, start on desktop, iterate on mobile, deploy from either.

### Product Delivery Revolution

Example: Describe app on commute → agents build full-stack → preview on phone → by evening desktop compiles native builds → agents generate App Store screenshots → publish → live in 1–2 days. Bridges the "Vibe Coding Gap."

## 7. Tokenomics & Incentive Economy

FORGE Token — ERC-20 (or SPL) utility token, earned through contribution (no pre-mine beyond initial airdrop and contributor allocation).

- **Earn**: building popular templates, maintaining connectors, community engagement.
- **Stake**: unlock premium features, higher rev share, governance.
- **Spend**: agent-to-agent payments, premium model access, white-label fees.
- **Burn**: 20% of net platform revenue (streams 1–7) used to buy back and burn FORGE.

### Allocation (1B total fixed)

- 44% Community Incentives & Ecosystem (4-year linear vest)
- 20% Core Contributors & Advisors (4-year + 1-year cliff)
- 15% Treasury & Foundation (DAO-controlled)
- 15% Investors (future strategic round, 3-year linear)
- 6% Initial Airdrop & Liquidity (immediate/3-month lock)

Proof of Value algorithm considers quality score, revenue generated, network growth, governance participation. Algorithm open-source and auditable.

## 8. Technical Architecture (Deep-Dive)

### High-Level Design

- **Client Layer**: Desktop IDE (Electron/Eclipse Theia), Web Creator Studio (Next.js), Mobile App (React Native)
- **API Gateway**: Kong/Envoy + WebSocket
- **Core Services**: Router Service (Rust), Agent Orchestrator (Rust, DAG), Deployment Pipeline (Terraform/Packer), Marketplace Service, Token Engine, Enterprise Governance, Computer Control (Desktop), Mobile Publishing Pipeline
- **Data Layer**: PostgreSQL, LanceDB (vector), Redis (cache), S3/MinIO (artifacts)
- **Infrastructure**: Kubernetes Cluster (Forge Cloud), Decentralized Compute Grid (Community GPUs), BYOC Gateways (AWS/GCP/Azure/DigitalOcean)

### Key Tech Stack

- Desktop IDE: Eclipse Theia (MIT license)
- Web: Next.js + React
- Mobile: React Native + native modules for App Store/Google Play
- CLI/Agent Runtime: Rust (clean-room)
- Backend API: TypeScript (Node.js) → progressively Rust
- Model Router: OpenRouter SDK + custom complexity classifier
- Local Memory: Vestige (SQLite + LanceDB)
- Deployment: Terraform/Packer + Caddy
- Orchestration: Kubernetes + Knative
- Blockchain: Ethereum L2 or Solana
- Smart Contracts: Solidity / Rust
- Computer Control: Platform-native APIs (AppleScript, Win32, xdotool) + screenshot vision loop

### Clean-Room Agent Harness

- Tool Registry: ~40 tools with permission gating
- Query Engine: all LLM calls with retry, cost tracking, circuit breaker
- Coordinator: DAG-based task decomposition, dynamic re-planning
- Memory: working, episodic (LanceDB), semantic (knowledge graph)
- KAIROS daemon: filesystem watcher → webhook → auto-fixes
- ULTRAPLAN: 30-min remote deep reasoning
- Auto Mode, Routines Engine

## 9. Process Flows (Concise)

**Non-Technical Founder (Mobile)**: Voice prompt → Product Strategy Agent → MVP definition → Agent swarm builds → preview on phone → tweak via voice → deploy to Forge Cloud → App Store publishing → lifecycle monitoring.

**Professional Developer (Desktop)**: Connect repo → Agent Canvas prompt → router selects model → agents fix bug, run tests, verify → PR opened → review → deploy to staging (BYOC) → background load test → push notification on success.

**Enterprise Outcome Contract**: Sign contract → requirements via Slack/Jira → compliance config → agents build in private VPC → UAT with verification swarm → deploy with full audit trail → monthly outcome review invoiced.

**Template Creator**: Build template → publish to marketplace → earn 70% revenue share + FORGE tokens per deployment → stake tokens for higher share.

**Self-Hosted User**: Download open-source agent runtime → install on own server → connect own API keys or local models → full functionality without Forge cloud.

**Token Economy**: Creator's template earns tokens → stake → buy template with FORGE → platform buys back and burns FORGE from revenue.

**Computer Control**: Agent opens Xcode, fixes alignment, rebuilds, takes screenshot, commits, notifies mobile.

**Mobile Publishing**: Agent generates metadata, screenshots, submits to App Store, user notified of review status.

## 10. Architectural Flow Diagrams

**Request Routing Flow**: User prompt → complexity classifier (on-device) → if trivial/simple → free-tier models (Llama, Gemma, etc.) → if moderate/complex → premium router (best quality or best cost) → fallback cascades → result to agent harness.

**Agent Swarm Coordination**: User goal → Product Strategy Agent → prioritized backlog → DAG coordinator decomposes into task graph with dependencies → verification swarm (security, QA, architecture) with 5-layer battery → deployment pipeline → live URL + lifecycle agent.

**Database & Backend Routing**: App created → backend strategy selection: simple → SQLite; full-stack → PostgreSQL (managed or user's cloud); enterprise → dedicated cluster in user's VPC; existing DB → connector.

## 16. Risk Register & Mitigations

- **API access blocked by providers**: OpenRouter abstraction, free-tier fallbacks, local model support
- **Competitors copy features**: data flywheel, token loyalty, BYOC ecosystem lock-in hard to replicate
- **Clean-room legal challenge**: established precedent (claw-code), different language (Rust)
- **Token volatility**: points first, stablecoin integration, buyback-and-burn
- **Deployment failures**: limit to known-good archetypes initially, expand with data
- **Computer control security**: app-level authorization, human-in-loop, sandboxing
- **Mobile App Store rejection**: follow guidelines, self-host alternative, transparency
