fix: per-provider key save, correct model filtering, thread recovery

- saveOneKey(provider, key): each Save button now saves ONLY that
  provider's key, not the entire apiKeys state. Prevents accidentally
  sending empty/stale keys for other providers.
- savedProviders state: populated from loadApiKeys() backend response
  (has_anthropic, has_openai etc) — source of truth for which providers
  are confirmed saved.
- Model dropdown: only shows models whose provider is in savedProviders.
  Claude key saved → Claude models appear. OpenAI key saved → GPT models.
- Thread recovery: if sendMessage gets THREAD_NOT_FOUND (Railway DB reset),
  auto-creates a new thread and retries the message seamlessly.
- loadApiKeys: no longer pollutes apiKeys state with __saved__ placeholders.
